Capitolo 6: Workflow e Pubblicazione

Un web-book non finisce quando il codice √® scritto. Il processo completo include testing, ottimizzazione, deployment e manutenzione.

Il Ciclo di Sviluppo

  1. Pianificazione: definire obiettivi e struttura
  2. Prototipazione: sketch e wireframe
  3. Sviluppo: scrivere HTML, CSS, JS
  4. Testing: verificare funzionalit√†
  5. Refinement: migliorare basandosi su feedback
  6. Deployment: pubblicare online
  7. Manutenzione: aggiornare contenuti

Testing Multi-Level

1. Testing Funzionale

  • Tutti i link funzionano?
  • Editor eseguono codice correttamente?
  • Soluzioni si mostrano/nascondono?
  • Navigazione prev/next corretta?

2. Testing Cross-Browser

Verificare su:

  • Chrome, Firefox, Safari, Edge
  • Versioni desktop e mobile
  • Diversi sistemi operativi

3. Testing Responsivo

Breakpoint critici:

  • Mobile: 320px - 480px
  • Tablet: 481px - 768px
  • Desktop: 769px+

4. Testing Accessibilit√†

  • Navigazione solo tastiera (Tab, Enter, Esc)
  • Screen reader (NVDA, JAWS, VoiceOver)
  • Contrasto colori con strumenti automatici
  • Validazione HTML (W3C Validator)

Deployment su GitHub Pages

Il workflow consigliato:

  1. Crea repository su GitHub
  2. Inizializza Git localmente
  3. Commit e push del codice
  4. Abilita GitHub Pages nelle impostazioni
  5. Sito live in pochi minuti!
# Comandi Git base
git init
git add .
git commit -m "Initial commit"
git remote add origin [URL]
git push -u origin main

Pro tip: Usa GitHub Actions per build automatiche e deployment continuo.

Manutenzione Continua

Dopo il lancio:

  • Monitora analytics: capire come gli utenti navigano
  • Raccogli feedback: form, email, social
  • Aggiorna contenuti: mantieni informazioni fresche
  • Fix bug: risolvi problemi segnalati
  • Migliora progressivamente: aggiungi feature richieste

Versionamento

Strategie per tracciare cambiamenti:

  • Git tags: marca versioni importanti
  • CHANGELOG.md: documenta ogni modifica
  • Semantic versioning: v1.2.3 (major.minor.patch)
  • Branch strategy: main/develop/feature

Metriche di Successo

Come misurare l'efficacia del web-book:

  • Engagement: tempo medio sulla pagina
  • Completamento: % utenti che finiscono il percorso
  • Interazione: uso editor, completamento esercizi
  • Retention: utenti che ritornano
  • Condivisione: link esterni e social shares
