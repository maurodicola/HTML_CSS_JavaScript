Capitolo 2: Architettura dell'Informazione

L'architettura dell'informazione (IA) è l'arte di organizzare contenuti in modo logico, trovabile e comprensibile. È il fondamento invisibile che rende un web-book usabile.

Struttura File e Cartelle

La struttura fisica riflette la struttura logica del contenuto: 

webbook/
├── index.html          → Homepage (porta d'ingresso)
├── capitolo-N.html     → Contenuti lineari
├── css/
│   └── style.css       → Presentazione separata
└── js/
    └── script.js       → Comportamento separato

Principio di Separazione delle Responsabilità

HTML → Struttura e contenuto
CSS → Presentazione e stile
JavaScript → Comportamento e interattività

Questa separazione facilita manutenzione, riutilizzo e collaborazione.

Naming Conventions

I nomi dei file seguono regole precise:

  • Minuscolo: capitolo-1.html non Capitolo-1.html
  • Trattini: come-creare.html non come_creare.html
  • Descrittivi: navigazione.html non nav.html
  • No caratteri speciali: solo a-z, 0-9, trattino

Perché? I sistemi Unix sono case-sensitive. Windows no, ma i server web sì. Usare sempre minuscolo previene errori di linking in produzione.

Gerarchia dei Contenuti

L'informazione è organizzata gerarchicamente:

  1. Sito (index.html) → panoramica completa
  2. Capitoli → temi principali
  3. Sezioni (h2) → argomenti nel capitolo
  4. Sottosezioni (h3, h4) → dettagli specifici

Modelli Mentali

Un buon web-book costruisce un modello mentale chiaro nella mente del lettore. Nell'Esercizio:

  • Homepage = punto di partenza, mappa completa
  • Capitoli = tappe progressive di un viaggio
  • Navigazione = binari che guidano il percorso
  • Esercizi = stazioni dove fermarsi e praticare

Progressione Logica

I capitoli dell'Esercizio seguono una progressione pedagogica:

Capitolo Fondamento Dipendenze
1. Introduzione Motivazione, contesto Nessuna
2. Struttura file Organizzazione Cap. 1
3. HTML Contenuto Cap. 1-2
4. CSS Stile Cap. 1-3
5. Navigazione Usabilità Cap. 1-4
6. JavaScript Interattività Cap. 1-5

Ogni capitolo costruisce sul precedente. Questa è architettura dell'informazione applicata.

Chunking (Segmentazione)

L'informazione complessa è "spezzata" in chunks digeribili. Regole applicate:

  • Un capitolo = un concetto principale
  • Una sezione = un sotto-concetto
  • Un paragrafo = un'idea singola
  • Lunghezza ottimale: 400-800 parole per sezione

Scannabilità

Il web si scansiona, non si legge. Design per la scansione:

  • Titoli descrittivi: sapere cosa aspettarsi
  • Liste puntate: informazioni in formato digeribile
  • Grassetto strategico: evidenziare concetti chiave
  • Box colorati: segnalare informazioni importanti
  • Spazi bianchi: dare respiro al contenuto

Findability (Trovabilità)

L'utente deve trovare ciò che cerca. Strategie:

  1. Indice nella homepage: panoramica completa
  2. Navigazione persistente: menu sempre visibile
  3. Link contestuali: riferimenti incrociati tra capitoli
  4. Breadcrumbs: "sei qui" navigation

Riepilogo

  • ✓ Struttura file riflette logica del contenuto
  • ✓ Naming conventions consistenti
  • ✓ Gerarchia chiara e progressiva
  • ✓ Chunking per comprensibilità
  • ✓ Design per scansione
  • ✓ Findability attraverso navigazione multipla
